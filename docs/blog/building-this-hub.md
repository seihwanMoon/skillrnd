# 이 사이트를 이렇게 키우는 이유

처음에는 `Codex` 기반 skill 문서를 한국어로 정리하는 작은 위키로 시작했다. 하지만 실제로는 단순한 스킬 목록보다 더 넓은 구조가 필요했다.

## 문제의식

- 좋은 skill과 도구가 너무 흩어져 있다.
- 글로벌 정보는 많지만 한국 실무 맥락 정리는 부족하다.
- 단순 링크 모음만으로는 실제 작업 흐름이 보이지 않는다.

## 그래서 택한 방향

이 사이트는 세 가지 역할을 동시에 수행한다.

1. 개인 블로그
2. skill 허브
3. 실전 아카이브

## 기대하는 결과

- 한국어 사용자도 빠르게 맥락을 이해할 수 있다.
- 도구 자체보다 `어떻게 조합해 쓰는지`가 드러난다.
- `vibe coding`, `제조 AI`, `한국 특화 실무`라는 세 축이 차별점을 만든다.

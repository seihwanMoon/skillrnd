# AI Coding Skills Wiki 프로젝트 계획서

## 1. 프로젝트 개요

### 1.1 배경

AI coding 도구와 agent 생태계가 빠르게 늘어나면서 `SKILL.md`, `AGENTS.md`, 레포지토리 템플릿, 도구 조합 예시가 흩어져 있다. 초기에 가장 실현 가능성이 높은 형태는 복잡한 SaaS가 아니라, 신뢰할 수 있는 정보를 한국어로 정리해 공개하는 정적 문서 사이트다.

### 1.2 목표

- AI coding 관련 `skills`를 한국어로 정리한 공개 지식 베이스 구축
- `설명`, `분류`, `사용 시점`, `구조`, `Mermaid`, `원문 링크`를 표준 문서 구조로 제공
- `GitHub Pages`로 저비용 운영
- 추후 `Codex` 기반 자동 수집 및 문서 생성 파이프라인으로 확장

### 1.3 비목표

- 로그인, 개인화, 데이터베이스 기반 서비스
- 사용자 입력을 받아 실시간 분석하는 동적 웹앱
- 첫 버전에서 모든 생태계의 skill 자동 수집 완성

## 2. 제품 정의

### 2.1 제품 형태

`VitePress` 기반 정적 문서 사이트

### 2.2 핵심 사용자

- AI coding 도구를 비교하고 싶은 개발자
- 에이전트 스킬 저장소를 빠르게 이해하고 싶은 사용자
- 한국어 문서가 필요한 팀 내부 학습자

### 2.3 핵심 가치

- 한국어로 빠르게 이해 가능
- 원문 링크와 구조 정보가 함께 제공됨
- 문서 형식이 일관되어 비교가 쉬움

## 3. MVP 범위

### 3.1 포함

- 홈 페이지
- 카테고리 허브
- 스킬 목록
- 대표 스킬 문서 3~10개
- 레시피 문서 2개 이상
- Mermaid 다이어그램 페이지
- GitHub Pages 배포 자동화

### 3.2 제외

- 자동 크롤러
- 검색 고도화
- 다국어
- 사용자 기여 워크플로우 자동화

## 5. 콘텐츠 스키마

개별 skill 문서는 아래 항목을 기본으로 유지한다.

1. 한줄 요약
2. 지원 대상 agent
3. 분류
4. 언제 쓰는가
5. 입력과 출력
6. 핵심 파일과 구조
7. 실행 예시
8. Mermaid 흐름도
9. 장점과 한계
10. 원문 링크
11. 관련 문서 링크

## 6. 구현 전략

### 6.1 1단계: 사이트 골격

- `VitePress` 설정
- 내비게이션과 사이드바 구성
- 커스텀 CSS 최소 적용
- GitHub Pages 워크플로우 추가

### 6.2 2단계: 콘텐츠 템플릿

- 카테고리 페이지 형식 확정
- skill 문서 템플릿 확정
- recipe 문서 템플릿 확정

### 6.3 3단계: 초기 콘텐츠 입력

- 우선순위 높은 대표 skill 입력
- 분류 기준 검증
- 다이어그램 작성

### 6.4 4단계: 반자동화

- Markdown 생성 스크립트 추가
- frontmatter 검증
- 링크 유효성 점검

### 6.5 5단계: 자동화 확장

- GitHub 저장소 입력
- `SKILL.md` 중심 파싱
- 한국어 요약 초안 자동 생성

## 8. 운영 규칙

- 사실과 해석을 구분한다.
- 문서마다 원문 링크를 남긴다.
- 불확실한 정보는 추정으로 표시한다.
- 문서 제목과 파일명은 안정적으로 유지한다.
- 한 문서 안에 너무 많은 다이어그램을 넣지 않는다.

## 9. 성공 기준

- 공개 가능한 첫 버전 완성
- GitHub Pages 자동 배포 성공
- 대표 skill 10개 이상 문서화
- 신규 skill 추가 시간이 30분 이내로 감소

## 10. 현재 결정

- 사이트 엔진: `VitePress`
- 배포: `GitHub Pages`
- 언어: 한국어 우선
- 운영 방식: 정적 문서 + 추후 자동화 확장

## 주요 목차별 상세 요약

### [00:00] 자연어 기반 시각화 생성 범위
- Claude 채팅 안에서 거의 모든 시각화를 바로 생성하는 흐름
- 별도 `/` 명령 없이 짧은 자연어 요청만으로 트리거
- 다이어그램, 인터랙티브 시뮬레이션, 사이트맵, 웹 구조까지 포괄
- Artifacts 계열 기능이 채팅 안으로 들어온 듯한 사용감
  - 근거와 예시
    - “diagram으로 보여줘”, “show it to me” 수준의 요청으로 작동
    - 별도 외부 툴 없이 채팅 내부에서 생성·수정·내보내기 언급
    - 예시 범주: 다이어그램, 시뮬레이션, 사이트맵, 블로그/브랜드 가이드
    - 사용 가능 범위로 어떤 subscription에서도 된다고 언급
  - 세부 요약
    - 기존에는 Figma 연결이나 napkin.ai 같은 외부 도구가 필요했다고 비교
    - 시각화 결과를 채팅에서 바로 확인하는 점을 핵심 가치로 제시
    - 마케팅, 블로그, SEO, 데이터셋, 레시피, 브랜드 가이드 등으로 활용처를 넓게 열어둠
    - 기능의 중심은 생성 자체보다 ‘채팅 안에서 바로 보고 고치는 흐름’

### [01:30] 복리 다이어그램의 실시간 상호작용
- 복리 개념을 인터랙티브 다이어그램으로 요청
- 변수 변경 즉시 반영되는 동작을 통해 시각화의 실용성 강조
- 생성 결과를 보며 실시간 작업 감각에 대한 만족감 표현
  - 근거와 예시
    - 예시 주제: compound interest
    - annual interest rate를 10%로 바꾸자 반영되는 장면
    - 변수 조정 후 작동 방식이 즉시 갱신된다고 서술
  - 세부 요약
    - 생성 후 바로 수치나 조건을 바꿔 결과를 다시 보는 흐름
    - 정적인 그림보다 조작 가능한 시각화의 장점을 보여줌
    - ‘AI가 실시간으로 일하는 모습’을 보는 재미를 강조
    - 실제 콘텐츠를 설명하는 데도 시뮬레이션형 출력이 유효함을 시사

### [03:00] 사이트맵 생성과 반복 보정
- 멜버른 배관업체 예시로 사이트맵 생성
- 서비스 4개를 SEO 구조로 묶어 시각화
- 첫 결과가 완벽하지 않아도 반복 지시로 개선 가능
- 같은 프롬프트라도 의도 해석이 어긋날 수 있어 추가 조건이 필요
  - 근거와 예시
    - 서비스 항목: emergency plumbing, pipe leak repair, hot water installation, commercial plumbing
    - 결과가 “not looking too bad” 수준으로 평가
    - building/shadow interaction 같은 추가 지시로 개선
    - 반복 뒤 “a little bit better”로 품질 향상 확인
  - 세부 요약
    - 사이트맵이 단순 목록이 아니라 구조화된 시각 결과로 나옴
    - SEO 최적화 구조라는 맥락이 명시됨
    - 생성 품질은 한 번에 고정되지 않고, 재지시를 통해 다듬는 방식
    - 정확히 같은 프롬프트라도 초기 결과가 기대와 다를 수 있다는 점을 드러냄

### [04:30] 내보내기와 실무 활용 흐름
- 완성물을 3점 메뉴에서 다운로드하거나 클립보드로 복사
- Google Docs에 바로 붙여넣는 사용 흐름
- 출력 형식은 작업 성격에 따라 SVG 또는 HTML로 달라짐
- 블로그, 페이지 구조, 브랜드 가이드 등 실무 배치 가능성 강조
  - 근거와 예시
    - 다이어그램은 SVG로 내려받는 사례
    - 시뮬레이션은 HTML로 내려받는 사례
    - 3점 메뉴에서 file download, clipboard copy 언급
    - Google Docs에 paste 가능하다고 설명
  - 세부 요약
    - 색상 변경은 내보내기 전에 해야 한다는 조건 제시
    - 블로그 삽입, 데이터셋 출력, 브랜드 가이드 반영 가능성 언급
    - 페이지를 어떻게 구조화할지, 블로그 글을 어떻게 보이게 할지에 특히 유용하다고 연결
    - 결과물을 단순 화면 확인이 아니라 실제 문서·웹 자산으로 옮기는 흐름을 보여줌

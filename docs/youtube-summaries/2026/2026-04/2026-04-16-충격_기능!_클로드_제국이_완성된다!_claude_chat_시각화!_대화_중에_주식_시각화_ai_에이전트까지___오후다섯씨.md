    - 텍스트로 설명받고 다시 다른 화면으로 이동하는 단계를 줄이는 방향
    - 클로드 채팅의 존재 이유를 다시 묻는 흐름에서 시각화가 핵심 근거로 제시

    - 그래프가 즉시 생성되고 하이라이트가 표시

    - 복리처럼 추상적인 개념도 채팅 안에서 시각화하면 이해 속도가 높아진다는 방향
    - 시각화는 단순 장식이 아니라 설명의 일부로 작동
    - 텍스트만으로는 느린 개념 전달을 보완하는 역할

    - 주식 분석을 대화형 대시보드로 바꾸는 것이 핵심
    - 종목명만 던지는 것이 아니라 판단 보조용 라벨을 덧붙이는 방식
    - 사용자는 검색과 정리를 동시에 받는 구조
    - 실제 투자 판단을 대체하기보다 탐색과 정리에 초점

    - 질문 자체가 기능 경계를 시험하는 성격
    - 영상 전체의 핵심 실험 축 중 하나

  - 세부 요약

    - 생산성은 단발성 결과보다 반복 가능성에서 나온다는 흐름

    - `교수님 계정 자동 인증` 미지원, `sandbox` 환경 제약
    - `Claude Code` 또는 별도 웹앱에서 구현해야 한다는 언급
  - 세부 요약
    - AI가 직접 외부 데이터를 읽고 진단하는 구조는 시연 환경에서 제한
    - 내장형 데모와 실제 서비스형 구현 사이의 간극을 드러냄
    - 기능의 가능성과 배포의 현실을 분리해서 보게 함

    - 아티팩트 기능으로 만드는 방식 자체의 언급
  - 세부 요약

    - AI가 붙어도 사용자 입장에서 결과물이 읽히는 방식이 중요
